How To Vend This Toolbox

This toolbox is designed to be copied into or used alongside target projects without dragging the whole repo in as an accidental runtime dependency.

Main Vend Paths

Full sidecar install: use install.py or sidecar_install when the target project should receive the complete `.dev-tools` working surface.
Thin shim install: use authority_install only when the legacy packed-authority path is the actual need.
Package copy: copy a package from packages/ into the target project when the whole subproject is the intended deliverable.
Template copy: copy from templates/ when the target only needs a document or starter reference surface.
python install.py
python src/tools/sidecar_install.py run --input-json "{\"target_project_root\": \"C:\\path\\to\\project\"}"
python src/tools/authority_install.py run --input-json "{\"target_project_root\": \"C:\\path\\to\\project\"}"
cp -r packages/_app-journal <target>/.dev-tools/_app-journal

What Not To Do

Do not make target applications silently depend on the live source of this repo. Vend the needed surface into the project, then let the project stand on its own.
