How Tools, Packages, And Templates Fit Together

The most important architectural move in this toolbox is choosing the correct surface before you start.

The Three Surfaces

Builder Tools

src/tools/ stays in the toolbox. Use these to inspect, patch, test, and package work on other projects.

Vendable Packages

packages/ contains subprojects that get copied into target projects when the project needs that capability.

Vendable Documents

templates/ contains reusable docs and starter reference material for new projects.

Decision Rule

If the capability should stay central and work on many projects, it belongs in a builder tool.
If the capability must live inside a target project, it belongs in a vendable package.
If the need is primarily doctrine or starter guidance, it belongs in a vendable document template.
