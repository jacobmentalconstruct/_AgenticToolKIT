How A New Session Starts

Cold starts are where drift begins. This toolbox reduces that by giving newcomers a short, repeatable orientation order.

Human Pickup Order

Open START_HERE.html for the tour-level explanation.
Read README.md for the full four-surface architecture.
Read CONTRACT.md and VENDORING.md to understand the boundaries.
Use the guide docs in _docs/ for the lived workflow and change trail.

Agent Pickup Order

Read toolbox_manifest.json first to know which tier fits the task.
Read tool_manifest.json to see the builder tools by name and category.
Read _docs/AGENT_GUIDE.md for the workflow loops and tool-selection logic.
Read _docs/EXPERIENTIAL_WORKFLOW.md for the collaboration rhythm behind the mechanics.

Why This Order Helps

The goal is to orient before building. The manifests answer “what exists,” the contract answers “what is allowed,” and the workflow docs answer “how this is actually used in practice.”
