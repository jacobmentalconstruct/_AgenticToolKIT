How The Workflow Feels In Practice

The raw docs explain the pieces. The lived workflow is the rhythm that ties them together: orient, choose the right surface, verify, document, and leave the next person a clean continuation point.

The Rhythm

Orient: manifests and docs first, not file wandering.
Select the right surface: builder tool, vendable package, or vendable document.
Build or vend deliberately: keep trust boundaries clean and avoid hidden coupling.
Verify: smoke tests, audits, or package-level checks before calling work complete.
Document: update the dev log or relevant docs so future sessions do not infer everything from diffs.
