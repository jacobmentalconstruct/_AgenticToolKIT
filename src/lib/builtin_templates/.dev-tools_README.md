# Dev Tools

This project uses `.dev-tools/` as its local builder utility surface.

## Expected Pattern

- vendored project authority lives under `.dev-tools/_project-authority/`
- agents should onboard from `.dev-tools/` before proposing structure changes
- human-readable doctrine is exported into `_docs/`

## Agent Start Rule

If you have low context, begin by:

1. checking `.dev-tools/`
2. finding `_project-authority/`
3. running the bootstrap/init path
4. reading the exported contract and onboarding docs
